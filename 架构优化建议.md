## 1. 当前架构问题分析

### 1.1 职责不清，耦合严重

| 文件 | 当前职责 | 问题 |
|------|----------|------|
| `TaskManager.py` | 业务逻辑 + 轴角变换 + 单位转换 | 数学计算不属于业务层 |
| `vision_to_robot.py` | 坐标转换 + 四元数运算 | 与 TaskManager 中数学实现重复 |
| `vision_trigger.py` | TCP通信 + 协议解析 | 缺乏统一通信抽象 |

### 1.2 缺乏工业级基础设施

- 无统一状态管理（设备状态机）
- 无统一异常处理和重试机制
- 配置分散，无配置中心
- 使用 `print()` 而非日志系统
- 缺乏安全联锁和异常兜底

### 1.3 机器人品牌耦合严重（关键问题）

当前架构直接依赖 UR5 机器人的 RTDE 协议和 `ur-rtde` 库，导致：
- 无法轻松替换为其他品牌机器人（如 Fanuc、KUKA、ABB、Epson 等）
- 业务层和服务层与特定机器人协议紧密耦合
- 测试和调试必须依赖真实的 UR5 硬件

### 1.4 耦合关系图

```
TaskApp.py
    │
    ├── TaskManager.py ───────────────────┐

    │       ├── RobotCtrl.py (UR5专用)     │ ← 直接依赖UR5协议
    │       ├── 轴角变换 (内嵌)            │

    │                                     │

    │                                     │

            │
            └── RobotCtrl.py (直接依赖)
```

## 2. 推荐架构方案：工业级分层架构

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        应用层 (Application Layer)                        │
│  TaskApp.py / GUI / CLI                                                 │
│  负责流程编排、用户交互、任务调度                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                        业务层 (Business Layer)                           │
│  TaskManager.py / ProcessController.py                                  │
│  封装业务逻辑、运动规划、工艺流程                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                        服务层 (Service Layer)                            │
│  MotionService.py / VisionService.py / CoordinateService.py             │
│  运动控制服务、视觉服务、坐标转换服务（业务无关）                          │
├─────────────────────────────────────────────────────────────────────────┤
│                    机器人抽象层 (Robot Adapter Layer)                     │
│  RobotInterface.py / RobotFactory.py / ur5_adapter.py / kuka_adapter.py │
│  统一机器人接口、适配器模式、工厂模式                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                        工具层 (Utility Layer)                            │
│  math_utils.py / config.py / logger.py / state_machine.py               │
│  数学计算、配置管理、日志、状态机、异常处理                                │
├─────────────────────────────────────────────────────────────────────────┤
│                        通信层 (Communication Layer)                      │
│  TCPClient.py / ProtocolParser.py / rtde_client.py / mc_client.py       │
│  RTDE/TCP/MC协议通信、协议解析、数据序列化                                │
├─────────────────────────────────────────────────────────────────────────┤
│                        硬件层 (Hardware Layer)                           │
│  UR5/KUKA/Fanuc/ABB 机器人 / 视觉系统 / IO设备                           │
│  实际硬件设备                                                           │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 设计思想

参考工业自动化领域主流架构模式：
- **ROS 模块化思想**：功能解耦，独立节点
- **PLC 分层架构**：控制层、逻辑层、接口层分离
- **工业互联网微服务理念**：服务化、可复用
- **适配器模式**：统一接口，适配不同品牌机器人
- **工厂模式**：动态创建不同品牌机器人实例

### 2.3 核心架构原则

| 原则 | 说明 |
|------|------|
| **接口抽象** | 定义统一的 `RobotInterface`，所有品牌适配器实现该接口 |
| **依赖倒置** | 高层模块依赖抽象接口，不依赖具体实现 |
| **开闭原则** | 对扩展开放，对修改关闭（新增品牌只需添加适配器） |
| **单一职责** | 每个模块只负责一个功能领域 |

#### 状态机设计 (`utils/state_machine.py`)

```python
from enum import Enum, auto

class RobotState(Enum):
    DISCONNECTED = auto()
    CONNECTED = auto()
    STANDBY = auto()
    RUNNING = auto()
    PAUSED = auto()
    ALARM = auto()
    EMERGENCY_STOP = auto()

class RobotStateMachine:
    def __init__(self):
        self.state = RobotState.DISCONNECTED
        self.transitions = {
            RobotState.DISCONNECTED: {RobotState.CONNECTED},
            RobotState.CONNECTED: {RobotState.STANDBY, RobotState.DISCONNECTED},
            RobotState.STANDBY: {RobotState.RUNNING, RobotState.DISCONNECTED},
            RobotState.RUNNING: {RobotState.PAUSED, RobotState.ALARM, RobotState.EMERGENCY_STOP},
            RobotState.PAUSED: {RobotState.RUNNING, RobotState.STANDBY},
            RobotState.ALARM: {RobotState.STANDBY},
            RobotState.EMERGENCY_STOP: {RobotState.STANDBY}
        }
    
    def transition(self, new_state):
        if new_state in self.transitions[self.state]:
            self.state = new_state
            return True
        raise ValueError(f"Invalid transition from {self.state} to {new_state}")
```

### 3.2 机器人抽象层（新增）

这是支持多品牌机器人的核心层，采用**适配器模式**和**工厂模式**。

#### 3.2.1 统一接口定义 (`robots/RobotInterface.py`)

```python
from abc import ABC, abstractmethod
from typing import List, Optional, Tuple

class RobotInterface(ABC):
    """
    机器人统一接口定义
    
    所有品牌的机器人适配器都必须实现此接口，确保上层服务层可以无缝切换不同品牌
    
    单位标准化契约（所有适配器必须遵守）:
    - 位置：米（m）
    - 角度：弧度（rad）
    - 线速度：米/秒（m/s）
    - 角速度：弧度/秒（rad/s）
    - 工具偏移：米（m）、弧度（rad）
    
    适配器内部必须处理品牌原生单位的转换：
    - UR：原生返回 m/rad，无需转换
    - Fanuc：原生返回 mm/deg，需转换为 m/rad
    - KUKA：原生返回 mm/deg，需转换为 m/rad
    - ABB：原生返回 mm/deg，需转换为 m/rad
    
    坐标系约定：
    - TCP 坐标系：默认与法兰坐标系重合（工具偏移为零时）
    - 工具偏移：工具坐标系相对于法兰坐标系的偏移量
    """
    
    @abstractmethod
    def connect(self) -> bool:
        """建立连接"""
        pass
    
    @abstractmethod
    def disconnect(self) -> None:
        """断开连接"""
        pass
    
    @abstractmethod
    def is_connected(self) -> bool:
        """检查连接状态"""
        pass
    
    @abstractmethod
    def get_tcp_pose(self) -> Optional[List[float]]:
        """获取TCP姿态 [x, y, z, rx, ry, rz]，单位：米、弧度"""
        pass
    
    @abstractmethod
    def get_joint_positions(self) -> Optional[List[float]]:
        """获取关节位置 [q1, q2, q3, q4, q5, q6]，单位：弧度"""
        pass
    
    @abstractmethod
    def get_tcp_speed(self) -> Optional[List[float]]:
        """获取TCP速度 [vx, vy, vz, vrx, vry, vrz]，单位：m/s、rad/s"""
        pass
    
    @abstractmethod
    def move_linear(self, target_pose: List[float], speed: float, 
                    acceleration: float) -> bool:
        """
        线性移动到目标位置
        
        参数:
            target_pose: [x, y, z, rx, ry, rz]，单位：米、弧度
            speed: 速度，单位：m/s
            acceleration: 加速度，单位：m/s²
        """
        pass
    
    @abstractmethod
    def move_joint(self, target_joints: List[float], speed: float, 
                   acceleration: float) -> bool:
        """
        关节空间移动到目标位置
        
        参数:
            target_joints: [q1, q2, q3, q4, q5, q6]，单位：弧度
            speed: 速度，单位：rad/s
            acceleration: 加速度，单位：rad/s²
        """
        pass
    
    @abstractmethod
    def stop(self) -> None:
        """停止运动"""
        pass
    
    @abstractmethod
    def get_robot_info(self) -> dict:
        """获取机器人信息（品牌、型号、版本等）"""
        pass
    
    @abstractmethod
    def get_error_code(self) -> Optional[int]:
        """获取错误码"""
        pass
    
    @abstractmethod
    def clear_error(self) -> bool:
        """清除错误"""
        pass
    
    @abstractmethod
    def set_tool_offset(self, offset: List[float]) -> bool:
        """
        设置工具偏移（工具坐标系相对于法兰坐标系的偏移）
        
        参数:
            offset: [tx, ty, tz, trx, try, trz]，单位：米、弧度
        
        返回:
            bool: 设置成功返回 True，失败返回 False
        """
        pass
    
    @abstractmethod
    def get_tool_offset(self) -> Optional[List[float]]:
        """
        获取当前工具偏移
        
        返回:
            [tx, ty, tz, trx, try, trz]，单位：米、弧度；如果未设置返回 None
        """
        pass
    
    @abstractmethod
    def format_state_for_vision(self) -> str:
        """
        格式化机器人状态数据，用于发送给视觉系统
        
        返回格式:
            "p,1,0,j1,j2,j3,j4,j5,j6,x,y,z,rx,ry,rz\\r"
        
        说明:
            - 关节角度 j1-j6：单位为度（deg）
            - 位置 x,y,z：单位为毫米（mm）
            - 旋转 rx,ry,rz：单位为度（deg）
            - 不同品牌适配器需根据视觉系统协议格式进行转换
        """
        pass
```

#### 3.2.2 工厂模式 (`robots/RobotFactory.py`)

```python
from typing import Optional
from .RobotInterface import RobotInterface
from .ur5_adapter import UR5Adapter
from .kuka_adapter import KUKAAdapter
from .fanuc_adapter import FanucAdapter
from .abb_adapter import ABBAdapter
from .simulation_adapter import SimulationAdapter
from utils.config import RobotBrand, RobotConfig

class RobotFactory:
    """
    机器人工厂类
    
    根据配置动态创建对应品牌的机器人适配器实例
    """
    
    @staticmethod
    def create(config: RobotConfig) -> Optional[RobotInterface]:
        """
        根据配置创建机器人实例
        
        参数:
            config: 机器人配置
        
        返回:
            机器人适配器实例，如果品牌不支持返回 None
        """
        brand = config.brand
        
        if brand == RobotBrand.UR:
            return UR5Adapter(config)
        elif brand == RobotBrand.KUKA:
            return KUKAAdapter(config)
        elif brand == RobotBrand.FANUC:
            return FanucAdapter(config)
        elif brand == RobotBrand.ABB:
            return ABBAdapter(config)
        elif brand == RobotBrand.SIMULATION:
            return SimulationAdapter(config)
        else:
            raise ValueError(f"不支持的机器人品牌: {brand}")
    
    @staticmethod
    def list_supported_brands() -> list:
        """列出支持的机器人品牌"""
        return [brand.value for brand in RobotBrand]
```

#### 3.2.3 UR5 适配器示例 (`robots/ur5_adapter.py`)

```python
from .RobotInterface import RobotInterface
from comm.rtde_client import RTDEClient
from utils.config import RobotConfig
from typing import List, Optional

class UR5Adapter(RobotInterface):
    """
    UR5 机器人适配器
    
    将 UR5 的 RTDE 协议接口适配到统一的 RobotInterface
    """
    
    def __init__(self, config: RobotConfig):
        self.config = config
        self.rtde_client = RTDEClient(config.ip, config.port)
        self._connected = False
    
    def connect(self) -> bool:
        self._connected = self.rtde_client.connect()
        return self._connected
    
    def disconnect(self) -> None:
        self.rtde_client.disconnect()
        self._connected = False
    
    def is_connected(self) -> bool:
        return self._connected
    
    def get_tcp_pose(self) -> Optional[List[float]]:
        if not self._connected:
            return None
        return self.rtde_client.get_actual_tcp_pose()
    
    def get_joint_positions(self) -> Optional[List[float]]:
        if not self._connected:
            return None
        return self.rtde_client.get_actual_q()
    
    def get_tcp_speed(self) -> Optional[List[float]]:
        if not self._connected:
            return None
        return self.rtde_client.get_actual_tcp_speed()
    
    def move_linear(self, target_pose: List[float], speed: float, 
                    acceleration: float) -> bool:
        if not self._connected:
            return False
        return self.rtde_client.move_l(target_pose, speed, acceleration)
    
    def move_joint(self, target_joints: List[float], speed: float, 
                   acceleration: float) -> bool:
        if not self._connected:
            return False
        return self.rtde_client.move_j(target_joints, speed, acceleration)
    
    def stop(self) -> None:
        if self._connected:
            self.rtde_client.stop()
    
    def get_robot_info(self) -> dict:
        return {
            'brand': 'UR',
            'model': 'UR5',
            'version': 'CB3/E-Series',
            'ip': self.config.ip
        }
    
    def get_error_code(self) -> Optional[int]:
        if not self._connected:
            return None
        return self.rtde_client.get_error_code()
    
    def clear_error(self) -> bool:
        if not self._connected:
            return False
        return self.rtde_client.clear_error()
    
    def set_tool_offset(self, offset: List[float]) -> bool:
        if not self._connected:
            return False
        return self.rtde_client.set_tcp(offset)
    
    def get_tool_offset(self) -> Optional[List[float]]:
        if not self._connected:
            return None
        return self.rtde_client.get_tcp()
    
    def format_state_for_vision(self) -> str:
        if not self._connected:
            return ""
        
        joints = self.get_joint_positions()
        pose = self.get_tcp_pose()
        
        if joints is None or pose is None:
            return ""
        
        # 转换单位：弧度→度，米→毫米
        j1, j2, j3, j4, j5, j6 = [math.degrees(j) for j in joints]
        x, y, z = [p * 1000 for p in pose[:3]]
        rx, ry, rz = [math.degrees(r) for r in pose[3:]]
        
        return f"p,1,0,{j1:.2f},{j2:.2f},{j3:.2f},{j4:.2f},{j5:.2f},{j6:.2f},{x:.2f},{y:.2f},{z:.2f},{rx:.2f},{ry:.2f},{rz:.2f}\r"
```

```python
        rx, ry, rz = [math.degrees(r) for r in pose[3:]]
        
        return f"p,1,0,{j1:.2f},{j2:.2f},{j3:.2f},{j4:.2f},{j5:.2f},{j6:.2f},{x:.2f},{y:.2f},{z:.2f},{rx:.2f},{ry:.2f},{rz:.2f}\r"
```

## 5. 核心设计原则

### 5.1 依赖注入

应用层创建依赖，通过构造函数注入到业务层和服务层：

```python
class TaskApp:
    def __init__(self, config):
        # 通过工厂创建机器人实例（依赖抽象）
        self.robot = RobotFactory.create(config.robot)
        self.motion_service = MotionService(self.robot)
        self.vision_service = VisionService(config.vision)
        self.task_manager = TaskManager(self.motion_service, self.vision_service)
```

### 5.2 适配器模式

通过适配器将不同品牌的机器人协议转换为统一接口：

```python
# 调用方代码（不关心具体品牌）
robot = RobotFactory.create(config)
robot.connect()
robot.move_linear([x, y, z, rx, ry, rz], speed, acceleration)

# 切换品牌只需修改配置
config.robot.brand = RobotBrand.KUKA
robot = RobotFactory.create(config)  # KUKA 适配器
robot.move_linear([x, y, z, rx, ry, rz], speed, acceleration)  # 接口完全相同
```

### 5.3 统一异常处理

定义分层异常体系，便于错误分类和处理：

```python
from utils.exceptions import RobotConnectionError, MotionFailedError, VisionError

try:
    self.motion_service.move_to_absolute(...)
except RobotConnectionError as e:
    logger.error(f"机器人连接失败: {e}")
    self.state_machine.transition(RobotState.DISCONNECTED)
except MotionFailedError as e:
    logger.error(f"运动失败: {e}")
    self.state_machine.transition(RobotState.ALARM)
```

### 5.4 安全联锁

在服务层实现安全检查机制：

```python
class MotionService:
    def move_to_absolute(self, pose, speed, acceleration):
        # 安全检查：速度范围、工作空间边界、碰撞检测
        if not self._validate_speed(speed):
            raise ValueError(f"速度超出安全范围: {speed}")
        if not self._check_workspace(pose):
            raise MotionFailedError("目标位置超出工作空间")
        # 执行移动（通过统一接口）
        return self.robot.move_linear(pose, speed, acceleration)
```

### 5.5 异步通信

使用 `asyncio` 实现非阻塞通信：

```python
import asyncio

class VisionService:
    async def trigger_async(self, robot_state):
        reader, writer = await asyncio.open_connection(
            self.config.host, self.config.port
        )
        command = self._build_command(robot_state)
        writer.write(command.encode())
        await writer.drain()
        response = await asyncio.wait_for(reader.read(1024), timeout=self.config.timeout)
        writer.close()
        await writer.wait_closed()
        return self._parse_response(response)
```

## 6. 多品牌机器人适配方案

### 6.1 适配器实现规范

每个品牌适配器必须遵循以下规范：

| 规范 | 说明 |
|------|------|
| **接口一致性** | 必须实现 `RobotInterface` 的所有方法 |
| **单位统一** | 位置使用米（m），角度使用弧度（rad），速度使用 m/s |
| **错误处理** | 返回 `None` 表示失败，不抛出品牌特定异常 |
| **连接状态** | 内部维护连接状态，`is_connected()` 必须准确 |
| **日志输出** | 使用统一的日志系统，添加品牌标识前缀 |

### 6.2 新增品牌适配流程

```
1. 在 config.py 中添加新品牌枚举值
2. 创建新的通信客户端（如 comm/KUKAClient.py）
3. 创建新的适配器（如 robots/kuka_adapter.py）
4. 在 RobotFactory.create() 中添加新品牌的创建逻辑
5. 创建单元测试（如 tests/test_kuka_adapter.py）
6. 修改配置文件，设置新品牌
```

### 6.3 各品牌协议特点

| 品牌 | 协议类型 | 通信方式 | 主要特点 |
|------|----------|----------|----------|
| **UR** | RTDE | TCP/IP | 实时数据交换，支持双向通信 |
| **KUKA** | EthernetKRL | TCP/IP | XML-RPC 风格，功能丰富 |
| **Fanuc** | FOCAS/EtherNet/IP | TCP/IP | 需额外许可证，功能强大 |
| **ABB** | MC3 / RobotStudio | TCP/IP | 基于 XML，支持高级功能 |
| **Epson** | EPSON RC+ | TCP/IP | 简单文本协议，易于集成 |

### 6.4 仿真模式支持

仿真适配器允许在没有真实硬件的情况下进行开发和测试：

```python
# 开发阶段使用仿真模式
config.robot.brand = RobotBrand.SIMULATION
robot = RobotFactory.create(config)

# 生产阶段切换到真实机器人
config.robot.brand = RobotBrand.UR
robot = RobotFactory.create(config)
```

## 8. 预期收益

### 8.1 量化对比

| 维度 | 当前状态 | 优化后 |
|------|----------|--------|
| **可维护性** | 耦合高，修改影响范围大 | 低耦合，单一职责 |
| **可扩展性** | 新增功能需修改多个文件 | 新增服务层即可 |
| **可靠性** | 缺乏异常处理和重试 | 完善的异常处理和重试机制 |

| **品牌适配性** | 硬编码 UR5，无法替换 | 支持多品牌切换，配置即可 |

1. **消除重复代码**：统一数学计算，避免 `TaskManager` 和 `vision_to_robot` 中的重复实现
2. **提升可靠性**：状态机管理、异常处理、重试机制
